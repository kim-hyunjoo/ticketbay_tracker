# 티켓베이 시세 트래킹 & 알림 시스템

## 프로젝트 개요

티켓베이 리셀 시장의 시세를 트래킹하고, 원하는 조건의 매물 발생 시 알림을 받으며, 과거 데이터 분석을 통해 매수/매도 타이밍을 판단할 수 있는 시스템.

---

## 1. 시세 트래킹

### 1-1. 트래킹 등록 및 관리

- 내가 등록한 공연(category_id)에 한해 시세 트래킹을 시작한다.
- 공연이 끝나면(막콘 기준) 트래킹을 자동 종료하고, 트래킹 대상에서 제외한다.
- 트래킹이 종료된 공연의 데이터는 삭제하지 않고 그대로 유지한다. (과거 패턴 분석용)

### 1-2. 트래킹 주기 관리

- 공연까지 남은 기간에 따라 트래킹 주기를 자동 조절한다.
  - D-30 이상: 1시간 간격
  - D-7 이내: 10분 간격
  - D-1: 5분 간격
- API 호출량 관리 및 차단 리스크 대응 목적.

### 1-3. API 상태 모니터링

- 티켓베이 API 응답 이상(비공식 API 스펙 변경, 차단 등)을 감지한다.
- 이상 발생 시 Discord 알림을 보낸다.

---

## 2. 공연 검색 및 등록

### 2-1. 검색을 통한 등록

- 가수 이름 또는 공연명으로 검색하면 티켓베이 검색 결과를 반환한다.
- 검색 결과에서 원하는 공연을 선택하여 트래킹 대상으로 등록할 수 있다.

### 2-2. category_id 직접 등록

- category_id를 직접 입력하여 트래킹 대상으로 추가할 수 있다.

### 2-3. (확장) 공연 공지 크롤링

- 관심 아티스트의 공식 채널(위버스 등)에서 공연 공지를 크롤링하여 새 공연 정보를 자동 감지한다.
- 우선순위: 낮음. 확장 가능하도록 구조만 고려.

## 3. 매물 알림 (바이어용)

### 3-1. 조건 기반 알림

- 원하는 조건을 등록하면, 해당 조건에 맞는 매물이 발생했을 때 Discord 알림을 보낸다.
- 조건 예시:
  - 350,000원 이하 + 스탠딩 + 400번대 이내
  - N원 이하 + 1층 + 6~11구역 + N열 이내
  - 원가 이하 (`list_price` 기준)

### 3-2. 복수 조건 세트 등록 (OR 조합)

- 여러 조건 세트를 OR 관계로 등록할 수 있다.
- 예: 스탠딩 35만 이하 OR 1층 앞구역 40만 이하 — 어느 쪽이든 매물이 뜨면 알림.

### 3-3. 알림 쿨다운 및 중복 방지

- 동일 매물에 대해 반복 알림이 가지 않도록 쿨다운을 설정한다.
- 이미 알림을 보낸 매물은 상태 변경(가격 변경 등)이 없는 한 재알림하지 않는다.

---

## 4. 시세 변동 알림 (셀러용)

### 4-1. 보유 티켓 기준 시세 알림

- 내가 보유한 티켓의 조건(구역, 열, 좌석 등)을 등록해둔다.
- 해당 조건의 시세가 특정 금액 이상으로 올랐을 때 알림을 보낸다.
- 매도 타이밍 판단 보조 목적.

## 5. 시세 시각화

### 5-1. 차트 공통 사항

- **X축은 D-day 기준으로 표시** (D-60, D-30, D-7, D-1)
  - 서로 다른 공연 간 시세 패턴을 동일 기준으로 비교 가능.
- **매물 수량은 각 차트에 보조 정보로 함께 표기**
  - 전체 최저가 차트: 해당 시점의 전체 매물 수량
  - 회차별 최저가 차트: 각 회차별 매물 수량
  - 구역별 최저가 차트: 각 구역별 매물 수량
  - (매물 50개에 최저가 30만원 vs 매물 3개에 최저가 30만원은 완전히 다른 상황)

### 5-2. 전체 회차 최저가 차트

- 각 D-day 기준으로 전 회차의 최저가를 보여준다.

### 5-3. 회차별 최저가 차트

- 각 D-day 기준으로 각 회차(날짜)의 최저가를 보여준다.

### 5-4. 구역별 최저가 차트

- 각 D-day 기준으로 구역별 최저가를 보여준다.
- 구역 구분은 티켓베이 매물 검색 필터의 좌석 위치 옵션 기준.
  - 예: KSPO돔 — 플로어, 1층, 2층
- **공연장별 구역 프리셋 관리** 필요 (우선순위: 중간)

### 5-5. 순수익 차트

- 매물 가격 기준 예상 순수익을 표시.
- **순수익 = 판매가 - (티켓베이 수수료 10% + 티켓 원가 + 티켓 수수료 + 배송비)**
- 티켓사별 기본값 예시 (멜론티켓): 수수료 4,000원, 배송비 3,700원
- 추후 티켓사별 기본값 설정 기능 추가 예정.
- 공연 간 비교 시 유용.

### 5-6. 거래 속도 / 매물 소진율

#### 측정 방식

- 트래킹 시점마다 매물 리스트를 스냅샷으로 저장, 이전 스냅샷과 비교:
  - **신규 매물**: 이전에 없었는데 새로 올라온 매물
  - **소진 매물**: 이전에 있었는데 사라진 매물 (판매 또는 등록 취소)
  - **잔존 매물**: 이전에도 있었고 지금도 남아있는 매물
- 매물 식별: 티켓베이 API 응답의 매물별 `id` 필드 사용 (확인됨)

#### 지표

| 지표               | 설명                                                    |
| ------------------ | ------------------------------------------------------- |
| 소진율             | 소진 매물 수 / 시작 시점 총 매물 수. 높으면 수요 > 공급 |
| 매물 회전율        | (신규 + 소진) / 평균 잔존 매물. 높으면 거래 활발        |
| 평균 생존 시간     | 등록 → 소진까지 걸린 시간. 짧아지면 올리면 바로 팔림    |
| 가격대별 소진 속도 | 원가 이하 / 원가~120% / 120% 이상 구간별 평균 소진 시간 |

#### 시각화 방식

- 기존 최저가 차트에 보조축(오른쪽 Y축) 또는 툴팁으로 일일 소진율, 신규 등록 수를 함께 표시.
- 매물 조회 뷰에서 매물별 등록/소진 시점 이력을 테이블로 조회 가능.

#### 활용 시나리오

- **셀러**: 소진율 상승 + 평균 생존 시간 감소 → 지금 올리면 빠르게 팔릴 가능성 높음.
- **바이어**: 소진율 높은 시기 → 알림 조건을 느슨하게 잡아야 매물을 놓치지 않음.

### 5-7. D-day 기반 시세 패턴 비교

- 같은 그룹의 이전 공연 데이터와 현재 공연을 D-day 기준으로 오버레이 비교.
- 지금이 고점인지 저점인지 감을 잡을 수 있다.

### 5-8. 그룹별 공연 시세 모아보기

- 각 아티스트별로 완료된/진행 중인 공연의 시세 차트를 모아서 볼 수 있다.

## 7. 티켓베이 API

### 엔드포인트

```
POST https://www.ticketbay.co.kr/ticketbayApi/product/v1/public/products
```

### Request Body

```json
{
  "category_id": 6437,
  "page": 0,
  "seat_floor": "스탠딩",
  "seat_grade": "1층",
  "seat_zone": "6구역"
}
```

- `category_id`: 공연별 고유 ID
- `page`: 0부터 시작, 페이지당 10개
- `seat_floor`, `seat_grade`, `seat_zone`: 필터 (단일 값만 지원, 다중 선택 불가)

### Response 구조

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "totalElements": 150,
    "totalPages": 15,
    "content": [
      {
        "id": 12345,
        "price": 350000,
        "list_price": 154000,
        "area": "스탠딩",
        "floor": "FLOOR",
        "grade": "",
        "zone": "",
        "perform_date": "2026-04-02",
        "sale_quantity": 1
      }
    ]
  }
}
```

### 주요 필드

| 필드            | 설명                              |
| --------------- | --------------------------------- |
| `id`            | 매물 고유 ID (소진율 추적에 사용) |
| `price`         | 매물 가격                         |
| `list_price`    | 원가 (프리미엄율 계산 기준)       |
| `area`          | 좌석 구역명                       |
| `floor`         | 층 (FLOOR, 1F, 2F 등)             |
| `perform_date`  | 공연 날짜                         |
| `sale_quantity` | 판매 수량                         |

## 12. Source of Truth 정책

- **코드가 단일 source of truth**
- Pen 파일: 디자인 탐색/커뮤니케이션 용도. 코드와 1:1 동기화를 강제하지 않음.
- SPEC.md: 결정 기록 (decision record). 기능 변경 시 함께 업데이트.
- PR 체크리스트: UI 변경 PR 시 pen 파일 업데이트 여부 확인.
- component-map.json: pen 노드 ID ↔ 코드 파일 매핑.
